# LaRa System Invariants

System invariants are non-negotiable architectural truths. These rules
must never be violated.

------------------------------------------------------------------------

## Invariant 1: LLM Is Not a Decision Engine

The LLM: - Generates language - Does not control difficulty - Does not
persist memory - Does not define mood

All adaptation is deterministic.

------------------------------------------------------------------------

## Invariant 2: Emotional Safety Over Intelligence

If a feature increases unpredictability, it must not be merged.

------------------------------------------------------------------------

## Invariant 3: No Emotional Labeling

LaRa must never: - Say "You are anxious." - Say "You are sad." - Create
identity-level statements.

LaRa responds to signals, not identities.

------------------------------------------------------------------------

## Invariant 4: Gradual Adaptation Only

-   No multi-level difficulty jumps
-   No oscillation
-   No instant resets
-   No task removal without structured transition

------------------------------------------------------------------------

## Invariant 5: Privacy Is Absolute

-   All processing local
-   No cloud emotional data
-   No biometric retention

------------------------------------------------------------------------

## Invariant 6: Predictability Is Required

-   One clear instruction at a time
-   No rapid escalation
-   No unpredictable novelty

------------------------------------------------------------------------

## Invariant 7: Micro-Challenge Must Always Exist

Tasks must never become trivial. Resilience requires controlled
challenge.

------------------------------------------------------------------------

## Final Principle

LaRa is not a chatbot.

LaRa is a regulated adaptive therapeutic system designed for
neurodiverse children.

Every system change must preserve: - Stability - Clarity - Safety -
Growth
